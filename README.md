# STDD — Specification & Test-Driven Development

STDD is a methodology for AI-assisted software development.

In STDD, the behavior of a system is defined by **specifications and tests**, while implementations become replaceable artifacts.

## The STDD Manifesto

- Specifications define intent.
- Tests verify behavior.
- Together, specifications and tests define the system.
- Implementations are replaceable artifacts.

## Conceptual Model

Specification → Tests → Implementation → Specification Fingerprint → Deploy
